📊 Baseline Model - Mathematical Formula / Modello Baseline - Formula Matematica

▼

Core Formula / Formula Base

λ_Home = μ × (OFF_home / μ) × (DEF_away / μ) λ_Away = μ × (OFF_away / μ) × (DEF_home / μ) Where / Dove: μ = League average corners per team / Media campionato corners per squadra OFF = Offensive strength (CK scored) / Forza offensiva (CK segnati) DEF = Defensive weakness (CK conceded) / Debolezza difensiva (CK subiti)

Context Weighting (NEW in V_5.3) / Pesatura Contestuale (NUOVO in V_5.3)

EN: Adaptively weights context-specific data (home/away) vs general performance.

IT: Pesa adattivamente i dati specifici del contesto (casa/trasferta) vs prestazione generale.

Weight = n_context / (n_context + K) OFF_final = w × OFF_context + (1-w) × OFF_general DEF_final = w × DEF_context + (1-w) × DEF_general K = Weight factor [1-15] / Fattore di peso [1-15] • K=1: Aggressive / Aggressivo (trust context with 1 match / fiducia al contesto con 1 partita) • K=5: Balanced / Bilanciato (50% weight at 5 matches / 50% peso a 5 partite) • K=15: Conservative / Conservativo (need many matches / servono molte partite)

Example Calculation / Esempio di Calcolo

EN: Team A plays at home with 20 total matches, 8 of which at home.

IT: Squadra A gioca in casa con 20 partite totali, 8 delle quali in casa.

Team A (Home / Casa): 20 total matches / partite totali, 8 home / casa
  OFF_general = 5.2 CK/match
  OFF_home = 5.8 CK/match
  K = 5
  
Weight = 8 / (8 + 5) = 0.615

OFF_final = 0.615 × 5.8 + 0.385 × 5.2 = 5.57 CK

League μ / Media campionato = 4.5 CK
Normalized / Normalizzato: 5.57 / 4.5 = 1.238

If / Se DEF_away = 1.10, then / allora:
λ_Home = 4.5 × 1.238 × 1.10 = 6.13 corners

🎯 Baseline V2 Model - Pure CK with Context Weighting / Modello Baseline V2 - CK Puro con Pesatura Contestuale

▼

✨ NEW in V_5.3 / NUOVO in V_5.3

EN: Baseline V2 is a mathematically rigorous corner prediction model that uses ONLY corner data (CK and CK_Against) with context-aware weighting. Unlike other models, it does NOT normalize with league averages, and home advantage is already implicit in the context-weighted data.

IT: Baseline V2 è un modello matematicamente rigoroso per la predizione dei corner che usa SOLO i dati corner (CK e CK_Against) con pesatura contestuale. A differenza di altri modelli, NON normalizza con le medie di campionato, e il vantaggio casa è già implicito nei dati pesati per contesto.

🎓 Core Philosophy / Filosofia di Base

Simplicity / Semplicità
EN: Uses only 2 statistics: CK (corners scored) and CK_Against (corners conceded). This eliminates multicollinearity and makes the model highly interpretable.
IT: Usa solo 2 statistiche: CK (corner segnati) e CK_Against (corner subiti). Questo elimina la multicollinearità e rende il modello altamente interpretabile.
Context-Awareness / Consapevolezza Contestuale
EN: Teams perform differently at home vs away. The model adaptively weights context-specific data (home/away matches) against general performance using the K parameter.
IT: Le squadre giocano diversamente in casa vs trasferta. Il modello pesa adattivamente i dati specifici del contesto (partite casa/trasferta) rispetto alla performance generale usando il parametro K.
Implicit Home Advantage / Vantaggio Casa Implicito
EN: No external home advantage multiplier is needed because the difference between home and away performance is already captured in the weighted averages.
IT: Non serve moltiplicatore esterno per il vantaggio casa perché la differenza tra performance casa e trasferta è già catturata nelle medie pesate.

📐 Mathematical Formulation / Formulazione Matematica

EN: The model works in 3 steps:

IT: Il modello opera in 3 passi:

STEP 1: Context Weighting / Pesatura Contestuale For each team, calculate context weight: w = n_context / (n_context + K) Where: - n_context = number of matches in specific context (home or away) - K = weight factor [1-15] (controls trust in context data) STEP 2: Weighted Averages / Medie Pesate Calculate offensive and defensive strengths: OFF_home = w_home × CK_home_context + (1 - w_home) × CK_home_general DEF_home = w_home × CKA_home_context + (1 - w_home) × CKA_home_general OFF_away = w_away × CK_away_context + (1 - w_away) × CK_away_general DEF_away = w_away × CKA_away_context + (1 - w_away) × CKA_away_general Where: - CK_context = avg corners in home/away matches only - CK_general = avg corners in all matches - CKA = CK_Against (corners conceded) STEP 3: Final Prediction with α Parameter / Predizione Finale con Parametro α λ_home = α × OFF_home + (1-α) × DEF_away λ_away = α × OFF_away + (1-α) × DEF_home Where: - α ∈ [0.4, 0.8] controls offense/defense weighting - α = 0.60 means 60% weight on offense, 40% on defense - α can be calibrated from historical data

⚙️ Il Parametro K (Weight Factor) / The K Parameter

🎯 Cosa fa K? / What does K do?

IT: K controlla quanto il modello si "fida" dei dati specifici del contesto (casa/trasferta) rispetto alla prestazione generale della squadra.

EN: K controls how much the model "trusts" context-specific data (home/away) versus the team's general performance.

Formula del Peso / Weight Formula: w = n_context / (n_context + K) Dove / Where: - n_context = numero di partite nel contesto specifico (es. 8 partite in casa) - K = fattore di peso [1-15] - w = peso finale dei dati contestuali [0-1] Esempio / Example: Se una squadra ha giocato 8 partite in casa: K=1 (aggressivo) → w = 8/(8+1) = 0.89 (89% peso ai dati casa) K=5 (bilanciato) → w = 8/(8+5) = 0.62 (62% peso ai dati casa) K=15 (conservativo) → w = 8/(8+15) = 0.35 (35% peso ai dati casa)

h4>⚖️ Il Parametro α (Alpha) - Bilanciamento Attacco/Difesa
λ = α × OFF + (1-α) × DEF_opponent Dove / Where: α ∈ [0.40, 0.80] = peso dato all'attacco vs difesa

IT: α determina quanto il modello si "fida" della capacità offensiva di una squadra rispetto alla debolezza difensiva dell'avversario.

EN: α determines how much the model "trusts" a team's offensive capability versus the opponent's defensive weakness.
